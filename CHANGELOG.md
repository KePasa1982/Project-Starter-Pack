# Changelog

All notable changes to **Project Starter Pack** (the factory repo) are summarized here. Child projects have their own history after bootstrap.

## [Unreleased]

- **Windows CI:** bootstrap resolves `git` / `npm` via `shutil.which` so `npm install` works when GitHub Actions runs Python on `windows-latest`.
- **Third-party attribution:** **[docs/THIRD_PARTY_NOTICES.md](docs/THIRD_PARTY_NOTICES.md)** for `karpathy-guidelines.mdc` (multica-ai/andrej-karpathy-skills, MIT); README section; copied to each child at bootstrap.
- **GitHub-ready factory:** removed local `template/**/node_modules` and `dist` artifacts; **[docs/PLATFORMS.md](docs/PLATFORMS.md)** (Linux / macOS / Windows prerequisites); **[docs/PUBLISHING.md](docs/PUBLISHING.md)** (maintainer checklist for first push); README platform pointer; privacy unit test **`TestFactoryRepoPrivacy`**; `.gitignore` adds `Thumbs.db`. Template **`vite.config.ts`** uses numeric port **5200** for factory **`npm run build`** / CI; bootstrap **`patch_vite_dev_port`** assigns each child’s unique port.
- **Premium bootstrap handoff:** **[docs/HANDOFF_MESSAGES.md](docs/HANDOFF_MESSAGES.md)** — Beat A/B user templates (checklist tables, dividers, icons, copy-friendly code blocks for path and `npm run dev`). Copied to each child at bootstrap; **`github-setup-offer.mdc`**, factory **AGENTS.md**, **README**, **WORKFLOW**, **GITHUB_SETUP_WALKTHROUGH**, **starter-pack-bootstrap**, **CURSOR_USER_RULE_SNIPPET** updated. Cursor **Copy** on fenced blocks — no fake HTML buttons.
- **GitHub push offer (ongoing):** template rule **`github-push-offer.mdc`** — after meaningful work, when `origin` exists, agents end with one line offering commit/push, commit only, or skip; never push without consent. Documented in **`AGENTS.md.template`**, **`FOR_USER.md`**, factory **README** / **AGENTS.md**, **`INDUSTRY_STANDARD_VERSIONING.md`**, **`GITHUB_SETUP_WALKTHROUGH.md`** Phase 5, **`UPGRADE_NOTES_CHILD_PROJECTS.md`**; tests in **`test_bootstrap_helpers.py`**.
- **Dev server host:** template `vite.config.ts` sets **`server.host: true`** so **`http://localhost:PORT`** and **`http://127.0.0.1:PORT`** both work (avoids IPv6-only `localhost` bind on some Linux setups). Bootstrap handoff URLs prefer **localhost** and note **127.0.0.1**.
- **Per-project dev port:** bootstrap assigns a unique Vite port (`5200–6099`) from the slug, skips ports used by sibling `psp-*` projects and ports in use on the machine, and writes it into `vite.config.ts` / README (`{{DEV_PORT}}`). Template uses **`strictPort: true`** and **`open: true`** so Cursor preview does not collide on `5173`.
- **Two-beat handoff:** Beat A = project created + GitHub **YES**/**NO** only; Beat B (after GitHub or **NO**) = GitHub result, local `npm run dev`, invite what to build. Bootstrap script no longer prints a long “What you do next” block for agents to paste at users.
- **GitHub setup after bootstrap:** agents ask *Let’s set up GitHub connection for this project?* (**YES** / **NO**) in the first handoff message; **[docs/GITHUB_SETUP_WALKTHROUGH.md](docs/GITHUB_SETUP_WALKTHROUGH.md)** (generic, no personal data in factory) copied to each child; template rule **`github-setup-offer.mdc`**; **AGENTS.md** / **README** / **WORKFLOW** / **CURSOR_USER_RULE_SNIPPET** updated.
- Removed static HTML remote-setup checklist, its `npm run …` script, and bootstrap JSON config for that page. Versioning guidance is consolidated in **[INDUSTRY_STANDARD_VERSIONING.md](INDUSTRY_STANDARD_VERSIONING.md)** (Git + optional remote; replaces the old factory-only guide file that focused on one host).
- Bootstrap: ignore `node_modules` / `dist` / `dist-ssr` / `.git` when copying; **`_materialize_with_cleanup`** (unit-tested) removes the folder if materialization fails before `git init`; same removal after failed `npm install` / `npm run build` / initial `git commit` post–`git init`; clearer **`--slug`** hint when title/slug contains non-ASCII; `under_skipped_dir` includes **`dist-ssr`**.
- CI: headless **`npm run dev`** smoke (curl) on fixed port; **npm fetch retry** env; matrix unchanged.
- Docs: [docs/UPGRADE_NOTES_CHILD_PROJECTS.md](docs/UPGRADE_NOTES_CHILD_PROJECTS.md) for legacy **`psp-…`** folders; **README** + **AGENTS** pointers; CONTRIBUTING **PowerShell** smoke + cleanup troubleshooting.
- Template: commit **`package-lock.json`** so **`npm ci`** works for a template-only checkout (matches child CI); every new child includes **[`.cursor/rules/karpathy-guidelines.mdc`](https://github.com/multica-ai/andrej-karpathy-skills)** (MIT) next to **`user-project-standard.mdc`** — see **`AGENTS.md.template`** / **`FOR_USER.md`**.
- Tests: `tests/test_bootstrap_helpers.py` — slug/title helpers, copytree ignore, materialize cleanup, **`TestChildTemplateCursorRules`** (Karpathy + `user-project-standard` in template, `AGENTS.md.template` links).
- Docs: **[CONTRIBUTING.md](CONTRIBUTING.md)** (local checks + changelog policy); README / FOR_USER wording avoids implying a fixed host path like `Projects/`.
- Factory **`.gitignore`** ignores `template/**/node_modules`, `dist`, and `dist-ssr` so local template dev never commits bulky artifacts.
