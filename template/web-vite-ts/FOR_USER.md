# For the user (this project)

This project was created from the **Project Starter Pack** so you do **not** need to manage folder layout or baseline backups yourself. The **`PSP` / `psp-`** prefix marks it as born from this pack.

**You:** describe what you want to create, try, or change — in plain language.

**Your agent:** keeps the repo organized, runs **`npm run build`**, and uses **Git**. Right after the project is created, they should ask whether you want **GitHub set up** (**YES** / **NO**) and walk you through it step by step if you say yes. After GitHub is linked, they may **remind you once per finished chunk of work** — e.g. *say if you want me to commit and push to GitHub* — but they will **not** push unless you agree. For optional off-machine backup, see **`docs/INDUSTRY_STANDARD_VERSIONING.md`**. Cursor loads **`.cursor/rules/`** (including [Karpathy-style coding habits](https://github.com/multica-ai/andrej-karpathy-skills)) so agents stay focused and avoid noisy refactors — you can delete `karpathy-guidelines.mdc` or `github-push-offer.mdc` if you do not want those layers. The first commit from bootstrap may use a placeholder git author until you **`git commit --amend --reset-author`**. **Never paste credentials or tokens into AI chat.**
